## 5 框架设计 Module Design

### 5.1 需求简要描述 Requirement description
*本节简要描述需求，包含输入输出、处理的功能及性能要求等。*

### 5.2 运行硬件环境 Runtime hardware environment
*本节描述系统运行需要的硬件环境，包括：CPU 性能、内存大小、硬盘容量、显示属性、通讯接口、人机操作设备等硬件配置。*

### 5.3 运行软件环境 Runtime software environment
*本节描述系统运行需要的软件运行环境，包括：操作系统、通讯协议栈、系统运行支持软件系统等软件配置。*

### 5.4 系统总体框架 Architecture
*本节按功能对系统进行模块池划分，给出系统的模块构造框图，定义划分的各个功能模块的名称，描述其将要完成的功能，并根据模块在系统框图中的位置说明模块间的调用关系。*

### 5.5 模块对需求支撑表 Block and requirement matrix
*以表格的形式描述各个模块对需求的支撑，表格可以使用需求项和功能项作为表格的横竖轴，在需求项和功能项的交叉位置进行标记。*

### 5.6 模块资源分配 Block resource
*详细描述每个模块运行资源分配，包含CPU占用、内存占用、响应延迟、存储占用等分配。*

### 5.7 继承性考虑(可选) Heritage feature (Optional)
*如果存在，本节描述设计对已有系统或相关系统已有技术和成果的继承方面的考虑。*

### 5.8 前瞻性考虑(可选) Advance feature (Optional)
*如果存在，本节描述对系统设计的前瞻性考虑，可以包括开发平台的平滑升级，运行平台的平滑升级、系统处理能力的提升、后续功能开发接口预留等方面。*
