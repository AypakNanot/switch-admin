# 智能交换机 Web 管理系统 - 架构设计文档

### Switch Web Management System - Architecture Design

## 1 目的 Purpose

本文档描述智能交换机 Web 管理系统的整体架构设计，包括：
- 系统模块划分与各模块功能定义
- 技术栈选型与运行环境
- 系统总体框架与模块间调用关系
- 数据结构与核心算法设计
- 数据库设计与可靠性设计
- 安全性设计与错误处理机制

本文档面向后端研发、前端研发、测试工程师，作为系统开发和集成的技术基准。

## 2 背景 Background

随着企业网络规模扩大，网络设备运维复杂度急剧上升。传统命令行配置方式存在以下痛点：
- 配置门槛高，需要熟记大量命令
- 误操作风险大，缺少可视化验证
- 故障定位慢，缺乏直观状态展示
- 批量操作效率低，重复劳动多

本项目基于 GoAdmin 框架，为 BroadEdge-S3652 系列智能交换机提供 Web 可视化管理系统，降低运维门槛，提升运维效率。

## 5 框架设计 Module Design

### 5.1 需求简要描述 Requirement description

**核心功能**：
- **配置模块**：端口、VLAN、STP、ERPS、PoE 等 15 个子模块配置管理
- **监控模块**：系统大盘、端口状态、流量监控
- **维护模块**：重启/保存、用户管理、系统配置、日志、SNMP、安全防护等 12 个子模块
- **网络模块**：IP 路由、诊断工具（Ping、Traceroute、线缆检测）

**性能要求**：
- API 响应时间 ≤ 100ms（读取缓存数据）
- 支持 10+ 并发用户
- 防 DDoS 刷新机制，避免 CPU 过载

### 5.2 运行硬件环境 Runtime hardware environment

**开发环境**：
- CPU：Intel Core i5 或同等性能
- 内存：≥ 8GB
- 存储：≥ 500MB 可用空间
- 网络：千兆以太网

**目标设备环境**（交换机嵌入式系统）：
- CPU：单核~双核 ARM/MIPS
- 内存：256MB ~ 512MB
- 存储：128MB ~ 512MB Flash
- 网络：10M/100M/1000M/10G 自适应

## 7 接口设计 Interface Design

### 7.1 用户接口 User Interface Design

**RESTful API 规范**：

| HTTP 方法 | 用途 | 示例 |
|-----------|------|------|
| GET | 查询资源 | GET /api/v1/ports |
| POST | 创建资源 | POST /api/v1/link-aggregation |
| PUT | 更新资源 | PUT /api/v1/ports/:port_id |
| DELETE | 删除资源 | DELETE /api/v1/link-aggregation/:id |

**统一响应格式**：

```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

**错误码定义**：

| 错误码 | 含义 | 处理方式 |
|--------|------|----------|
| 200 | 成功 | - |
| 400 | 参数错误 | 返回具体错误字段 |
| 403 | 权限不足 | 提示用户无权限 |
| 404 | 资源不存在 | 自动刷新列表 |
| 409 | 冲突 | 返回冲突详情 |
| 500 | 系统错误 | 记录日志，提示重试 |

### 7.2 外部接口 Extern Interface Design

### 8.2 核心算法 Algorithm Design

#### 8.2.1 模式切换算法

```
算法：SwitchMode(oldMode, newMode)
输入：旧模式 oldMode, 新模式 newMode
输出：错误 error

流程：
1. 验证 newMode 是否有效
   IF !newMode.IsValid() THEN
       RETURN ErrInvalidMode
   END IF

2. 获取写锁
   mutex.Lock()
   DEFER mutex.Unlock()

3. 检查是否相同
   IF oldMode == newMode THEN
       RETURN nil  // 无需切换
   END IF

4. 持久化到数据库
   IF configDAO != nil THEN
       err = configDAO.SetRunMode(newMode.String())
       IF err != nil THEN
           RETURN err
       END IF
   END IF

5. 更新内存
   currentMode = newMode

6. 触发模式切换事件（TODO）
   // 清理旧资源、初始化新组件

7. RETURN nil
```

#### 8.2.2 端口统计偏移量算法

```
算法：GetPortStatsWithOffset(portID)
输入：端口 ID portID
输出：相对统计值 PortStats

背景：硬件计数器是全局共享的，不能直接清零
      采用偏移量机制计算相对值

流程：
1. 读取硬件计数器
   hwStats = ReadHardwareCounters(portID)
   // rx_bytes, tx_bytes, rx_packets, tx_packets, ...

2. 读取偏移量
   offset = LoadOffset(portID)
   // 从 /var/run/port_stats_offset.json 加载
   // 如果不存在，offset = 0

3. 计算相对值
   relativeStats.rx_bytes = hwStats.rx_bytes - offset.rx_bytes
   relativeStats.tx_bytes = hwStats.tx_bytes - offset.tx_bytes
   relativeStats.rx_packets = hwStats.rx_packets - offset.rx_packets
   relativeStats.tx_packets = hwStats.tx_packets - offset.tx_packets
   ...

4. 处理溢出（64 位环绕）
   IF relativeStats.rx_bytes < 0 THEN
       relativeStats.rx_bytes += UINT64_MAX + 1
   END IF
   ...

5. RETURN relativeStats

清零操作：
函数：ClearPortStats(portID)
流程：
1. hwStats = ReadHardwareCounters(portID)
2. offset = hwStats  // 将当前值设为偏移量
3. SaveOffset(portID, offset)
4. // 下次读取时，relativeStats = hwStats - offset = 0
```

#### 8.2.3 异步任务管理算法

```
算法：CreateDiagnosticTask(taskType, params)
输入：任务类型 taskType, 参数 params
输出：任务 ID taskID

流程：
1. 生成任务 ID
   taskID = GenerateTaskID(taskType, timestamp)

2. 创建任务记录
   task = &Task{
       TaskID:     taskID,
       Type:       taskType,
       Params:     params,
       Status:     "running",
       CreatedAt:  time.Now(),
       WebSocketConnected: true,
       TimeoutSec: GetTimeout(taskType),  // Ping:30s, Traceroute:120s
   }

3. 存储任务
   taskStore[taskID] = task

4. 启动后台 goroutine 执行任务
   go executeTask(task)

5. RETURN taskID

轮询查询：
算法：GetTaskResult(taskID)
流程：
1. task = taskStore[taskID]
2. IF task == nil THEN
       RETURN 404 Not Found
   END IF
3. IF task.Status == "running" THEN
       RETURN {status: "running", progress: task.Progress}
   END IF
4. IF task.Status == "completed" THEN
       RETURN {status: "completed", result: task.Result}
   END IF
5. IF task.Status == "failed" THEN
       RETURN {status: "failed", error: task.Error}
   END IF

清理孤儿任务：
算法：CleanupOrphanTasks()
周期：每 60 秒执行

流程：
1. FOR each task IN taskStore DO
       // 检查 WebSocket 连接
       IF !task.WebSocketConnected THEN
           elapsed = time.Since(task.CreatedAt)
           IF elapsed > task.TimeoutSec THEN
               TerminateTask(task)
               REMOVE task FROM taskStore
           END IF
       END IF

       // 检查执行超时
       IF task.Status == "running" THEN
           elapsed = time.Since(task.CreatedAt)
           IF elapsed > task.TimeoutSec THEN
               TerminateTask(task)
               task.Status = "failed"
               task.Error = "timeout"
           END IF
       END IF
   END FOR
```
