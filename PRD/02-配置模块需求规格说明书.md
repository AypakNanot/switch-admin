## 一、模块概述

### 1.1 功能定位

配置模块是交换机 Web 管理系统的核心模块，提供二层网络功能的配置能力，包括端口状态、链路聚合、风暴控制、流量控制、端口隔离、端口监测、VLAN、VLAN分类、MAC、生成树、ERPS、PoE、端口镜像、组播、资源、堆叠等。本模块具有**高互斥性**和**强约束性**特点，不当配置可能导致网络故障。

### 1.2 用户场景与价值

**典型用户**：网络运维工程师

**核心场景**：
1. **端口管理**：配置端口状态、速率、双工、描述等基础参数
2. **链路聚合**：配置 LACP 聚合组提升带宽和可靠性
3. **安全防护**：配置端口安全、风暴控制、端口隔离防止网络攻击
4. **VLAN划分**：创建 VLAN、配置端口 VLAN 模式和成员关系
5. **环路防护**：配置生成树、ERPS 防止二层环路

**业务价值**：
- 提供可视化配置界面，降低配置门槛
- 实时校验配置约束，防止错误配置
- 支持批量操作，提升运维效率

### 1.3 权限要求

| 功能 | 权限要求 |
|------|----------|
| 所有配置功能 | admin/operator |
| 配置查看 | admin/operator/viewer |

### 1.4 配置互斥约束总表（关键！）

```
┌─────────────────────────────────────────────────────────────────────────────────────────────┐
│ ⚠️ 配置互斥约束一览表                                                                       │
├─────────────────────────────────────────────────────────────────────────────────────────────┤
│ 端口模式互斥：                                                                               │
│ ├─ Access 模式：必须设置 PVID，不能设置 Allowed VLAN，不能设置 Tagged VLAN                 │
│ ├─ Trunk 模式：必须设置 PVID 和 Allowed VLAN，可设置 Tagged VLAN                           │
│ └─ Hybrid 模式：可灵活设置 PVID、Untagged VLAN、Tagged VLAN                                │
│                                                                                             │
│ 功能互斥：                                                                                   │
│ ├─ 端口加入聚合组后，不能单独配置速率/双工/流控（由聚合组统一管理）                         │
│ ├─ 端口开启端口安全后，不能配置静态 MAC 地址（由安全学习机制管理）                          │
│ ├─ 端口配置 802.1X 认证后，端口安全功能受限                                                 │
│ ├─ 端口配置 ERPS 环网后，生成树协议在该端口自动禁用                                         │
│ └─ 端口配置为镜像源端口后，不能同时作为镜像目的端口                                         │
│                                                                                             │
│ 参数约束：                                                                                   │
│ ├─ PVID 必须在端口允许的 VLAN 列表中                                                        │
│ ├─ Allowed VLAN 不能包含保留 VLAN（1, 4095）                                                │
│ ├─ 聚合组 ID 范围：1-128（部分设备 1-64）                                                   │
│ ├─ 生成树优先级必须为 4096 的倍数（0, 4096, 8192, ... 61440）                               │
│ └─ MAC 地址学习上限：0-4096（0 表示不限制）                                                 │
│                                                                                             │
│ ⚠️ 高危操作防护机制（Network-Killing 防护）：                                               │
│ ├─ 【自杀式切断防护】修改端口 VLAN 模式/PVID 时，若可能导致管理连接中断，必须弹出红色读秒警告│
│ ├─ 【聚合继承法则】端口加入聚合组后，独立配置被挂起，移出后恢复原配置或出厂默认            │
│ ├─ 【PoE 实际状态】区分管理状态与运行状态，功率不足时显示 Power Denied 及原因              │
│ └─ 【堆叠分裂惩罚】双主冲突时，备机业务端口强制 Error-Down，防止 MAC 风暴和 IP 冲突        │
└─────────────────────────────────────────────────────────────────────────────────────────────┘
```

### 3.5 成员端口继承法则（基因覆盖 Assimilation）

**⚠️ 核心问题**：物理端口加入聚合组后，其独立配置如何处理？

**继承法则**：

```
┌─────────────────────────────────────────────────────────────────────────────────────────────┐
│ ⚠️ 成员端口继承法则                                                                         │
├─────────────────────────────────────────────────────────────────────────────────────────────┤
│ 加入聚合组时：                                                                               │
│ ├─ 物理端口的独立配置（VLAN、流控、风暴控制、QoS）被【挂起保存】                           │
│ ├─ 物理端口强制【继承】聚合组的配置                                                         │
│ └─ 界面上物理端口显示 ⚠️ 标记，提示"配置由聚合组管理"                                       │
│                                                                                             │
│ 移出聚合组时：                                                                               │
│ ├─ 物理端口【恢复】加入前的独立配置                                                         │
│ ├─ 若加入前无配置，恢复为出厂默认配置                                                       │
│ └─ 界面上 ⚠️ 标记消失，端口恢复独立管理                                                    │
└─────────────────────────────────────────────────────────────────────────────────────────────┘
```

```
│                                                                                             │
│                              [取消] [确认加入]                                              │
└─────────────────────────────────────────────────────────────────────────────────────────────┘
```

**配置保存与恢复机制**：

| 配置项 | 加入聚合组时 | 移出聚合组时 |
|--------|--------------|--------------|
| VLAN 模式/PVID/Allowed | 挂起保存，继承聚合组配置 | 恢复原配置 |
| 流量控制 | 挂起保存，继承聚合组配置 | 恢复原配置 |
| 风暴控制 | 挂起保存，继承聚合组配置 | 恢复原配置 |
| QoS 策略 | 挂起保存，继承聚合组配置 | 恢复原配置 |
| 端口安全 | 必须先禁用才能加入聚合组 | 保持禁用状态 |
| 端口镜像 | 必须先移除镜像配置 | 保持无镜像状态 |

```
### 13.3 PoE实际状态说明（沉默罢工防护）

**⚠️ 核心问题**：PoE 管理状态为"启用"不等于实际在供电，功率不足、设备故障都会导致"启用但未供电"的假象。

**实际状态枚举**：

| 状态图标 | 状态名称 | 说明 | 处理建议 |
|----------|----------|------|----------|
| 🟢 | 供电中(Powering) | 正常供电，设备正常运行 | - |
| 🟡 | 检测中(Detecting) | 正在检测受电设备 | 等待检测完成 |
| 🟡 | 分类中(Classifying) | 正在对受电设备分类 | 等待分类完成 |
| 🔴 | 功率不足(Power Denied) | 系统剩余功率不足以支持该设备 | 降低其他端口功耗或升级PoE电源 |
| 🔴 | 过载保护(Overload) | 受电设备功耗超过端口最大功率 | 检查设备功耗或提高端口最大功率 |
| ⚫ | 短路保护(Short Circuit) | 检测到短路，端口已保护关闭 | 检查线缆和设备，修复后恢复 |
| ⚫ | 未供电(Disabled) | 管理状态为禁用 | 启用PoE |
| ⚫ | 设备断开(No Device) | 管理启用但未检测到受电设备 | 检查设备连接 |
```

```
### 17.7 分裂惩罚机制（MAD - Multi-Active Detection）

**⚠️ 核心风险**：堆叠分裂（堆叠线断开）会导致网络中出现两台"完全相同"的交换机（相同的 IP、MAC、配置），造成毁灭性的 MAC 漂移和 IP 冲突。

**MAD 惩罚动作定义**：

```
┌─────────────────────────────────────────────────────────────────────────────────────────────┐
│ ⚠️ 堆叠分裂惩罚机制（MAD Action）                                                           │
├─────────────────────────────────────────────────────────────────────────────────────────────┤
│ 触发条件：                                                                                   │
│ 堆叠链路断开，导致原堆叠系统分裂为多个独立运行的子系统                                       │
│                                                                                             │
│ 检测方式：                                                                                   │
│ ├─ 基本检测：通过堆叠端口状态检测                                                           │
│ └─ 增强型检测：通过专用检测链路（直连线或业务端口）更快检测                                 │
│                                                                                             │
│ 惩罚动作（分裂发生后立即执行）：                                                             │
│ ┌───────────────────────────────────────────────────────────────────────────────────────┐   │
│ │ 1. 角色裁决                                                                            │   │
│ │    ├─ 比较各子系统的主交换机优先级                                                     │   │
│ │    ├─ 优先级最高者成为合法主（Master）                                                 │   │
│ │    └─ 其他子系统的原主/备交换机降级为"分裂备机"（Split-Slave）                        │   │
│ │                                                                                       │   │
│ │ 2. 端口隔离（Error-Down）                                                              │   │
│ │    ┌─────────────────────────────────────────────────────────────────────────────┐   │   │
│ │    │ 合法主（Master）                                                            │   │   │
│ │    │ └─ 所有业务端口：保持正常运行                                                │   │   │
│ │    │                                                                              │   │   │
│ │    │ 分裂备机（Split-Slave）                                                      │   │   │
│ │    ├─ 业务端口：全部强制置为 Error-Down（关闭）                                   │   │   │
│ │    ├─ 保留端口：管理口、检测口、Console 口                                        │   │   │
│ │    └─ 效果：该设备完全隔离，防止 MAC 飘移和 IP 冲突                               │   │   │
│ │    └─────────────────────────────────────────────────────────────────────────────┘   │   │
│ │                                                                                       │   │
│ │ 3. 告警触发                                                                            │   │
│ │    ├─ 系统日志：记录 Emergency 级别告警                                              │   │
│ │    ├─ 系统告警：首页大盘红色闪烁提示                                                 │   │
│ │    └─ SNMP Trap：发送堆叠分裂告警通知                                                │   │
│ └───────────────────────────────────────────────────────────────────────────────────────┘   │
│                                                                                             │
│ 恢复流程：                                                                                   │
│ ├─ 自动恢复：堆叠链路恢复后，分裂备机自动重启并重新加入堆叠                                │
│ └─ 手动恢复：管理员手动检查后，通过命令恢复分裂备机的业务端口                              │
└─────────────────────────────────────────────────────────────────────────────────────────────┘
