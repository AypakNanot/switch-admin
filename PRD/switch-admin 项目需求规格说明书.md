## 1. 执行摘要 (Executive Summary)

### 1.1 问题陈述

网络运维人员需要频繁使用命令行配置和监控交换机，操作门槛高、易出错，且缺乏可视化的状态展示，导致运维效率低下、故障定位时间长。

### 1.2 解决方案

开发基于 Web 的智能交换机管理系统，通过可视化界面替代命令行操作，提供端口配置、状态监控、系统维护和网络诊断等核心功能，降低运维门槛，提升运维效率。

### 1.3 成功标准

| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| 配置效率提升 | ≥ 50% | 对比命令行与 Web 界面完成相同配置任务的时间 |
| 运维错误率降低 | ≥ 80% | 统计误操作次数 |
| 故障 MTTR 缩短 | ≥ 40% | 对比平均故障修复时间 |
| 页面响应时间 | ≤ 100ms | Dashboard 等核心页面的 API 响应时间 |
| 系统可用性 | ≥ 99.9% | 系统正常运行时间占比 |

|------|------|----------|

### 2.3 验收标准 (Acceptance Criteria)

#### AC-系统大盘
- [ ] Dashboard 展示产品型号、软件版本、运行时间、序列号
- [ ] Dashboard 展示端口总数、活跃端口数、Down 端口数、端口利用率
- [ ] Dashboard 展示端口状态列表（端口、状态、链路、速率、双工、描述）
- [ ] 数据刷新延迟 ≤ 5 秒

#### AC-端口监控
- [ ] 支持按状态（启用/禁用）、链路（Up/Down）筛选
- [ ] 支持单端口统计清零
- [ ] 支持全局统计清零（需二次确认）
- [ ] 流量统计使用 64 位无符号整数

#### AC-用户管理
- [ ] 支持创建、编辑、删除用户
- [ ] 支持设置用户角色（admin/operator/viewer）
- [ ] 权限校验正确，不能越权操作

#### AC-网络诊断
- [ ] Ping 支持设置目标 IP、次数、超时时间
- [ ] Traceroute 支持设置目标 IP、最大跳数
- [ ] 诊断任务支持取消

### 2.4 非目标 (Non-Goals)

以下功能**不在**当前版本范围内：

| 功能 | 说明 |
|------|------|
| 流量趋势历史图表 | 需要额外存储和计算资源 |
| 高级告警系统 | 需要邮件/短信集成 |
| 多设备集群管理 | 当前仅支持单设备管理 |
| 自动化运维脚本 | 需要额外的脚本引擎 |
| 第三方系统集成 | 需要额外的 API 开发 |

## 3. 系统需求 (System Requirements)

### 3.1 技术架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        用户浏览器                                │
│                    http://localhost:9033/admin                   │
└───────────────────────────┬─────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│                      Go + Gin Web 服务器                        │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────────┐ │
│  │  路由层     │  │  处理器层   │  │    服务层               │ │
│  │  (Gin)      │→ │  (Handler)  │→ │    (Service)            │ │
│  └─────────────┘  └─────────────┘  └─────────────────────────┘ │
│                            │                     │              │
│                            ▼                     ▼              │
│                   ┌─────────────────┐  ┌─────────────────┐     │
│                   │   GoAdmin 框架  │  │   SQLite 数据库 │     │
│                   │  (AdminLTE 主题) │  │   (admin.db)    │     │
│                   └─────────────────┘  └─────────────────┘     │
└─────────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│                      交换机底层驱动                              │
│            (SNMP / CLI / 系统调用 / 硬件寄存器)                   │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **后端框架** | Go + Gin | 高性能、并发友好 |
| **管理框架** | GoAdmin | 快速构建后台管理系统 |
| **UI 主题** | AdminLTE | 成熟的企业级 UI 组件 |
| **数据库** | SQLite3 | 轻量级嵌入式数据库 |
| **部署方式** | 单二进制文件 | 便于部署和分发 |

### 3.3 运行环境约束

| 约束项 | 规格 | 影响 |
|--------|------|------|
| **设备内存** | 256MB ~ 512MB | 严禁大数据量内存驻留 |
| **CPU** | 单核 ~ 双核 ARM/MIPS | 限制并发处理能力 |
| **存储** | 128MB ~ 512MB Flash | 限制配置写入频率 |
| **网络带宽** | 10M/100M/1000M/10G | 计数器需 64 位防溢出 |

### 3.4 集成点 (Integration Points)

| 集成点 | 接口方式 | 说明 |
|--------|----------|------|
| **交换机硬件** | 系统调用/驱动 | 获取端口状态、配置参数 |
| **SNMP** | SNMP v2/v3 | 可选的监控和管理协议 |
| **文件系统** | 本地文件系统 | 日志、配置文件存储 |
| **数据库** | SQLite3 | 用户、配置持久化 |

### 3.5 安全与隐私 (Security & Privacy)

| 安全要求 | 实现方式 |
|----------|----------|
| **用户认证** | Session/Cookie 认证 |
| **权限控制** | 基于角色的访问控制 (RBAC) |
| **输入校验** | 前后端双重校验，防注入攻击 |
| **会话管理** | 会话超时、并发会话限制 |
| **日志审计** | 记录所有配置变更操作 |

## 5. 风险与挑战 (Risks & Challenges)

### 5.1 技术风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| 硬件驱动兼容性问题 | 高 | 中 | 提前与硬件团队对接，获取驱动文档 |
| 性能瓶颈（大端口数） | 中 | 中 | 数据聚合缓存、分页加载 |
| 64 位计数器溢出 | 高 | 高 | 严格使用 uint64_t，API 传输用字符串 |
| 并发配置冲突 | 高 | 中 | 配置版本号、乐观锁机制 |

### 5.2 项目风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| 需求变更频繁 | 高 | 中 | 建立变更管理流程，评估影响 |
| 硬件资源受限 | 中 | 高 | 提前进行性能测试，优化资源使用 |
| 测试环境不足 | 中 | 中 | 使用模拟器 + 真实设备结合测试 |

### 5.3 已知漏洞与改进建议

#### 漏洞 #1: 配置回滚机制缺失
- **问题**: 误操作后只能手动恢复或重启
- **建议**: 实现配置快照和一键回滚功能

#### 漏洞 #2: 并发配置冲突检测不足
- **问题**: 多用户同时修改可能覆盖配置
- **建议**: 实现配置版本号和乐观锁机制
