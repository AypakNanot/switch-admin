# 📋 全局错误码总表

**文档版本**：1.0  
**创建日期**：2025-03-12  
**适用范围**：智能交换机 Web 管理系统 - 所有模块

## 📖 使用说明

本文档整合了所有模块的错误码定义，按**模块分类**和**错误类型**组织。

**快速查找**：
- 知道模块 → 直接跳转到对应模块章节
- 知道错误类型 → 查看 HTTP 状态码分类

---

## 一、HTTP 状态码分类

### 通用状态码（所有模块适用）

| HTTP 状态码 | 错误码 | 说明 | 处理建议 |
|:-----------:|--------|------|----------|
| 400 | INVALID_PARAM | 参数校验失败 | 检查请求参数格式和类型 |
| 400 | PARAM_MISSING | 必填参数缺失 | 补充缺失的必填参数 |
| 403 | PERMISSION_DENIED | 权限不足 | 联系管理员提升权限 |
| 404 | RESOURCE_NOT_FOUND | 资源不存在 | 检查资源 ID 是否正确 |
| 409 | CONCURRENT_CONFLICT | 并发冲突 | 等待并发操作完成后重试 |
| 409 | CONFIG_CONFLICT | 配置冲突 | 检查配置互斥约束 |
| 429 | RATE_LIMIT_EXCEEDED | 请求过于频繁 | 降低请求频率 |
| 500 | INTERNAL_ERROR | 系统内部错误 | 联系技术支持，提供错误日志 |
| 503 | RESOURCE_INSUFFICIENT | 系统资源不足 | 释放资源或稍后重试 |
| 504 | DEVICE_TIMEOUT | 设备响应超时 | 检查设备连接状态，重试 |

## 七、错误响应格式规范

### 7.1 标准错误响应格式

```json
{
  "code": 404,
  "error": "VLAN_NOT_FOUND",
  "message": "VLAN 不存在",
  "details": {
    "vlan_id": 100,
    "suggestion": "检查 VLAN ID 是否在 1-4094 范围内"
  },
  "timestamp": "2025-03-12T10:30:00Z",
  "request_id": "req_abc123xyz"
}
```

### 7.2 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | HTTP 状态码 |
| error | String | 错误码（英文大写 + 下划线） |
| message | String | 错误描述（中文，面向用户） |
| details | Object | 可选，详细错误信息和建议 |
| timestamp | String | ISO 8601 格式的时间戳 |
| request_id | String | 请求 ID，用于日志追踪 |

### 7.3 并发冲突特殊响应

当发生并发冲突时，返回当前配置状态：

```json
{
  "code": 409,
```

## 八、错误码命名规范

### 8.1 命名规则

- **格式**：`[模块前缀]_[资源类型]_[错误类型]`
- **大小写**：全大写，单词间用下划线分隔
- **模块前缀**：可选，用于区分模块（如 PORT、VLAN、LAG）

### 8.2 常见错误类型后缀

| 后缀 | 说明 | 示例 |
|------|------|------|
| NOT_FOUND | 资源不存在 | VLAN_NOT_FOUND |
| EXISTS | 资源已存在 | USER_EXISTS |
| INVALID | 参数无效 | PASSWORD_INVALID |
| CONFLICT | 配置冲突 | PORT_MODE_CONFLICT |
| FAILED | 操作失败 | CONFIG_SAVE_FAILED |
| EXCEEDED | 数量超限 | MAC_TABLE_FULL |
| INSUFFICIENT | 资源不足 | RESOURCE_INSUFFICIENT |
| TIMEOUT | 操作超时 | DEVICE_TIMEOUT |
| DENIED | 权限拒绝 | PERMISSION_DENIED |

### 8.3 错误码分配原则

1. **唯一性**：每个错误码在全局范围内唯一
2. **语义清晰**：错误码应清晰表达错误原因
3. **可扩展**：预留错误码空间供未来扩展
4. **向后兼容**：已发布的错误码不得删除或修改语义
