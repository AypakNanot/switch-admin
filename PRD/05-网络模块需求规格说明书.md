## 二、IP路由

### 用户场景与价值

**典型用户**：网络管理员

**核心场景**：
1. **路由规划**：管理员需要为新接入的业务网段配置静态路由，确保流量能够正确转发到目标网络。
2. **故障排查**：当网络出现不通时，管理员需要查看路由表，确认是否存在正确的路由条目、路由是否被正确学习。
3. **路由优化**：管理员需要调整路由优先级（距离值），确保流量走最优路径。

**业务价值**：
- 提供可视化的路由配置能力，降低命令行操作的门槛和误操作风险
- 缩短路由故障排查时间（MTTR）
- 支持多协议路由的统一查看，便于网络拓扑分析

**前置条件**：
- 用户已登录系统，且具有「网络配置」相关权限（详见权限约束章节）
- 交换机各接口已正确配置 IP 地址

### 2.1 IPv4路由表

**功能说明**：路由信息查询功能，用于查看交换机当前的路由表，了解所有路由条目及其来源（静态配置或动态协议学习）。

#### 2.1.1 界面布局

**菜单路径**：网络 → IP路由 → IPv4路由表

```
┌─────────────────────────────────────────────────────────┐
│ IPv4路由表                                               │
├─────────────────────────────────────────────────────────┤
│ 目的IP   [________________]  协议类型 [▼ 全部    ]      │
│                          [查询]                         │
├─────────────────────────────────────────────────────────┤
│ 目的IP地址 │ 目的IP掩码 │ 协议类型 │ 出端口              │
├─────────────────────────────────────────────────────────┤
│                    暂无数据~                             │
└─────────────────────────────────────────────────────────┘
```

**界面优化说明**：

| 控件 | 说明 | 设计理由 |
|------|------|----------|
| 目的IP搜索框 | 支持 IP 地址模糊搜索和最长前缀匹配 | 核心交换机可能有上万条路由，仅靠协议类型筛选无法快速定位。支持输入 "10.10.25." 查找所有匹配路由 |

**搜索模式说明**：

| 模式 | 示例输入 | 匹配逻辑 |
|------|----------|----------|
| 精确匹配 | 192.168.1.0 | 查找目的IP = 192.168.1.0 的路由 |
| 模糊匹配 | 10.10. | 查找目的IP 以 10.10. 开头的所有路由 |
| 最长前缀匹配 | 10.10.25.30（非网段地址） | 查找能够匹配该 IP 的最精确路由条目 |

#### 2.1.2 字段定义

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 目的IP地址 | IPv4 | 目标网段地址 |
| 目的IP掩码 | String | 子网掩码 |
| 协议类型 | Enum | 路由来源：Static(静态)/OSPF/RIP/BGP/Connected(直连)/Local(本地) 等 |
| 出端口 | String | 数据包出口，如 vlan1、eth0/1 |

#### 2.1.3 数据量级与分页策略

**问题背景**：核心交换机的路由表可能包含数千甚至数万条路由条目（如 BGP 全量路由）。全量返回会导致：
- 交换机 CPU/内存压力过大
- 网络传输超时
- 前端渲染卡顿

**分页策略**：

| 参数 | 值 | 说明 |
|------|-----|------|
| 默认每页条数 | 50 | 平衡性能与用户体验 |
| 最大每页条数 | 200 | 防止恶意请求 |
| 最大返回条数 | 5000 | 无分页时硬限制 |
| 排序规则 | 按 协议类型 → 目的IP地址 升序 | 便于快速定位 |

**API 分页参数**：

```
GET /api/v1/routes/table?dest_ip={ip}&protocol={type}&page=1&page_size=50
```

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| dest_ip | String | 否 | 目的IP地址，支持模糊匹配和最长前缀匹配 |
| protocol | String | 否 | 协议类型筛选，为空返回全部 |
| page | Integer | 否 | 页码，默认 1 |
| page_size | Integer | 否 | 每页条数，默认 50，最大 200 |

**响应示例**：

```json
{
  "code": 200,
  "data": {
    "total": 12580,
    "page": 1,
    "page_size": 50,
    "total_pages": 252,
    "items": [
      {
        "dest_ip": "0.0.0.0",
        "dest_mask": "0.0.0.0",
        "protocol": "Static",
        "out_port": "vlan1"
      }
    ]
  }
}
```

#### 2.1.4 交互逻辑

**主流程（Happy Path）**：
1. 进入页面，默认显示全部协议类型的路由，第1页数据
2. 用户可通过「协议类型」下拉框筛选特定类型
3. 点击「查询」按钮刷新列表，重置为第1页
4. 无数据时显示"暂无数据~"

**异常流程（Exception Flow）**：

| 异常场景 | 处理方式 |
|----------|----------|
| 查询超时（>30s） | 提示"查询超时，路由表数据量较大，请稍后重试" |
| 交换机未响应 | 提示"设备无响应，请检查网络连接" |
| 权限不足 | 隐藏该菜单项，或显示"无权限访问"提示页 |

**状态同步机制**：

| 策略 | 选择 | 说明 |
|------|------|------|
| 自动刷新 | ❌ 不启用 | 路由表变化频率较低，避免不必要的资源消耗 |
| 手动刷新 | ✅ 启用 | 用户点击「查询」按钮刷新 |
| 页面停留超时 | 提示 | 页面停留超过 5 分钟后，顶部显示"数据可能已过期，建议点击查询刷新" |

#### 2.1.5 权限约束

| 用户角色 | 查看权限 | 备注 |
|----------|----------|------|
| admin | ✅ | 完全可见 |
| operator | ✅ | 只读权限，可查看但不可操作 |
| viewer | ✅ | 只读权限 |

### 2.2 IPv4静态路由信息

**功能说明**：路由配置功能，用于管理交换机的静态路由，包括新建、编辑、删除路由条目。

#### 2.2.1 列表页界面

**菜单路径**：网络 → IP路由 → IPv4静态路由信息

```
┌─────────────────────────────────────────────────────────┐
│ IPv4静态路由信息                                          │
├─────────────────────────────────────────────────────────┤
│ [新建] [删除]                                            │
├─────────────────────────────────────────────────────────┤
│ 目的IP地址 │ 目的IP掩码 │ 路由距离 │ 操作              │
├─────────────────────────────────────────────────────────┤
│                    暂无数据~                             │
└─────────────────────────────────────────────────────────┘
```

#### 2.2.2 列表页字段定义

| 字段名 | 类型 | 长度 | 必填 | 默认值 | 校验规则 | 说明 |
|--------|------|------|------|--------|----------|------|
| 目的IP地址 | IPv4 | 15字符 | 是 | - | 合法单播IPv4 | 目标网段或主机地址 |
| 目的IP掩码 | String | 18字符 | 是 | - | 下拉选择 | 子网掩码，显示格式为"掩码(位数)" |
| 路由距离 | Integer | 3位 | 否 | 1 | 范围：1~255 | 管理距离/优先级 |
| 操作 | - | - | - | - | - | 编辑/删除按钮 |

#### 2.2.3 列表页交互逻辑

**主流程（Happy Path）**：

| 操作 | 流程 |
|------|------|
| 新建 | 点击「新建」→ 弹出/跳转配置表单 → 填写保存 → 返回列表，显示新增记录 |
| 编辑 | 点击「编辑」→ 弹出/跳转配置表单（预填数据）→ 修改保存 → 返回列表，显示更新记录 |
| 删除 | 选中记录 → 点击「删除」→ 二次确认弹窗 → 确认删除 → 刷新列表 |
| 批量删除 | 勾选多条记录 → 点击「删除」→ 二次确认弹窗 → 确认删除 → 刷新列表 |

**异常流程（Exception Flow）**：

| 异常场景 | 处理方式 |
|----------|----------|
| 删除时路由已被其他进程修改 | 提示"该路由配置已变更，请刷新后重试"，自动刷新列表 |
| 删除核心路由（如默认路由 0.0.0.0/0） | 弹窗警告"删除默认路由可能导致网络中断，确认继续？" |
| 批量删除超过 50 条 | 提示"单次最多删除 50 条，请分批操作" |

**权限与按钮状态**：

| 用户角色 | 新建 | 编辑 | 删除 | 按钮显示 |
|----------|------|------|------|----------|
| admin | ✅ | ✅ | ✅ | 正常显示 |
| operator | ❌ | ❌ | ❌ | 按钮置灰，hover 提示"无操作权限" |
| viewer | ❌ | ❌ | ❌ | 按钮隐藏 |

#### 2.2.4 配置表单界面

**页面标题**：IPv4静态路由配置

```
┌─────────────────────────────────────────────────────────┐
│ IPv4静态路由配置                                          │
├─────────────────────────────────────────────────────────┤
│ *目的IP地址    [________________] (0.0.0.0)              │
│ *目的IP掩码    [▼ 255.255.255.0(24)    ]                │
│  下一跳       [________________]                       │
│  路由距离     [___] (1~255, 默认 1)                     │
├─────────────────────────────────────────────────────────┤
│              [应用]  [返回]                              │
└─────────────────────────────────────────────────────────┘
```

#### 2.2.5 表单字段定义

| 字段 | 控件类型 | 必填 | 默认值 | 说明 |
|------|----------|------|--------|------|
| 目的IP地址 | 文本输入 | 是 | - | placeholder 显示 0.0.0.0 |
| 目的IP掩码 | 下拉选择 | 是 | - | 格式：掩码值(位数)，如 255.255.255.0(24) |
| 下一跳 | 文本输入 | 否 | - | 下一跳路由器地址 |
| 路由距离 | 数字输入 | 否 | 1 | 管理距离，值越小优先级越高 |

#### 2.2.6 表单校验规则（前端实时校验）

**目的IP地址校验**（失焦触发）：

| 校验项 | 规则 | 错误提示 |
|--------|------|----------|
| 格式校验 | 合法 IPv4 格式 | "请输入有效的 IPv4 地址" |
| 单播地址 | 不能是 0.x.x.x（网络地址） | "目的IP不能为网络地址" |
| 禁用环回 | 不能是 127.x.x.x | "目的IP不能为环回地址" |
| 禁用组播 | 不能是 224.0.0.0 ~ 239.255.255.255 | "目的IP不能为组播地址" |
| 禁用广播 | 不能是 255.255.255.255 | "目的IP不能为广播地址" |

**目的IP掩码校验**：

| 校验项 | 规则 | 错误提示 |
|--------|------|----------|
| 必填校验 | 必须选择 | "请选择子网掩码" |
| 合法性 | 必须是连续的 1 和 0 | "子网掩码格式不正确" |

**下一跳校验**（失焦触发，仅当填写时校验）：

| 校验项 | 规则 | 错误提示 |
|--------|------|----------|
| 必填校验 | 可为空（表示直连路由或策略路由场景） | - |
| 格式校验 | 合法 IPv4 格式 | "请输入有效的 IPv4 地址" |
| 单播地址 | 不能是组播/广播/环回地址 | "下一跳必须为有效单播地址" |
| 禁止全零 | 不能是 0.0.0.0 | "下一跳不能为 0.0.0.0" |
| 本地接口冲突 | 不能与交换机本地接口 IP 相同 | "下一跳不能为本机接口地址" |

**路由距离校验**：

| 校验项 | 规则 | 错误提示 |
|--------|------|----------|
| 范围校验 | 1 ~ 255 的整数 | "路由距离范围为 1-255" |

#### 2.2.7 表单提交逻辑

**主流程（Happy Path）**：

```
1. 用户点击「应用」
2. 前端执行全部校验规则
3. 校验通过 → 按钮变为"保存中..."，禁用按钮
4. 调用后端 API
5. 后端返回 200 → 提示"静态路由配置成功" → 关闭表单 → 刷新列表
```

**异常流程（Exception Flow）**：

| 异常场景 | HTTP状态码 | 错误码 | 处理方式 |
|----------|------------|--------|----------|
| 目的网段已存在 | 409 | ROUTE_EXISTS | 提示"该目的网段路由已存在，请勿重复配置" |
| 下一跳不可达 | - | - | **允许保存**，但列表中显示⚠️警告图标，hover提示"下一跳当前不可达，路由暂未生效"。详见"递归路由说明" |
| 底层下发失败 | 500 | CHIP_ERROR | 提示"路由表下发失败，设备可能资源不足" |
| 网络超时 | - | TIMEOUT | 提示"请求超时，请检查网络连接后重试" |
| 并发冲突 | 409 | CONCURRENT_MODIFY | 提示"配置已被其他用户修改，请刷新后重试" |

**递归路由（下一跳不可达）说明**：

| 项目 | 说明 |
|------|------|
| 概念 | 静态路由的下一跳地址不在本机直连网段，需要通过其他路由（如默认路由）到达下一跳 |
| 业务场景 | 例如：目的网段 192.168.10.0/24，下一跳 10.0.0.1，但本机没有到 10.0.0.1 的直连路由 |
| 处理策略 | **允许配置保存**（可能后续会配置到达下一跳的路由），但在列表中标记警告 |
| 列表显示 | ⚠️ 图标 + tooltip "下一跳 [10.0.0.1] 当前不可达，该路由暂未生效" |
| 状态刷新 | 当系统检测到下一跳变为可达时，自动移除警告标记 |

**列表字段扩展（递归路由支持）**：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 状态 | Enum | ✅ 生效 / ⚠️ 下一跳不可达 / ❌ 配置错误 |
| 状态说明 | String | 仅在状态非"生效"时显示，说明原因 |

**防重复提交**：
- 点击「应用」后，按钮立即进入 Loading 状态
- 按钮文字变为"保存中..."
- 禁用按钮，直到 API 返回结果或超时（30秒）

#### 2.2.8 API 结构设计

```
GET    /api/v1/routes/static              # 获取静态路由列表
POST   /api/v1/routes/static              # 新建静态路由
GET    /api/v1/routes/static/{id}         # 获取单条静态路由详情
PUT    /api/v1/routes/static/{id}         # 更新静态路由
DELETE /api/v1/routes/static              # 批量删除静态路由
DELETE /api/v1/routes/static/{id}         # 删除单条静态路由
```

**新建请求示例**：

```json
// POST /api/v1/routes/static
{
  "dest_ip": "192.168.10.0",
  "dest_mask": "255.255.255.0",
  "next_hop": "10.10.25.1",
  "distance": 1
}
```

**成功响应**：

```json
{
  "code": 200,
  "message": "静态路由配置成功",
  "data": {
    "id": "route_001",
    "dest_ip": "192.168.10.0",
    "dest_mask": "255.255.255.0",
    "next_hop": "10.10.25.1",
    "distance": 1
  }
}
```

**错误响应示例**：

```json
// 路由已存在
{
  "code": 409,
  "error": "ROUTE_EXISTS",
  "message": "该目的网段路由已存在，请勿重复配置"
}

// 下一跳不可达
{
  "code": 400,
  "error": "NEXTHOP_UNREACHABLE",
  "message": "下一跳地址不可达，请检查与本机接口是否同网段"
}
```

## 三、网络诊断工具

### 用户场景与价值

**典型用户**：网络管理员、运维工程师

**核心场景**：
1. **连通性验证**：配置完路由后，需要验证与目标网络是否连通。
2. **故障定位**：当业务系统无法访问时，需要逐跳排查网络路径，定位故障节点。
3. **线路检测**：怀疑网线物理故障时，需要检测电缆状态（断路、短路、线序错误）。

**业务价值**：
- 提供可视化诊断工具，降低命令行操作门槛
- 缩短故障排查 MTTR（平均修复时间）
- 支持多 VRF 实例诊断，适应多租户网络环境

**权限要求**：
- 所有角色均可使用诊断工具（只读操作，不影响设备配置）
- 但需注意：频繁执行诊断可能消耗设备 CPU 资源

---

### 3.1 Ping

**功能说明**：网络连通性测试工具，通过发送 ICMP Echo 请求并接收响应，判断与目标主机之间的网络是否通畅、延迟情况如何。

#### 3.1.1 界面布局

**菜单路径**：网络 → Ping

```
┌─────────────────────────────────────────────────────────┐
│ Ping 参数                                                │
├─────────────────────────────────────────────────────────┤
│ VRF ID      [▼ mgmt vrf     ]                           │
│ 目标IP地址  [________________] (192.168.1.1)            │
├─────────────────────────────────────────────────────────┤
│                        [Ping]                           │
├─────────────────────────────────────────────────────────┤
│ (结果显示区域)                                           │
│ PING 192.168.1.1 (192.168.1.1): 56 data bytes           │
│ 64 bytes from 192.168.1.1: seq=1 ttl=64 time=1.23ms     │
│ 64 bytes from 192.168.1.1: seq=2 ttl=64 time=1.45ms     │
│ --- 192.168.1.1 ping statistics ---                     │
│ 4 packets transmitted, 4 received, 0% packet loss       │
│ round-trip min/avg/max = 1.12/1.34/1.56 ms              │
└─────────────────────────────────────────────────────────┘
```

#### 3.1.2 字段定义

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| VRF ID | Enum | 是 | mgmt vrf | 虚拟路由转发实例，选项：mgmt vrf、default vrf |
| 目标IP地址 | IPv4/域名 | 是 | - | 要测试的目标地址，支持 IP 或域名 |

#### 3.1.3 输入校验规则

**目标IP地址校验**（失焦触发）：

| 校验项 | 规则 | 错误提示 |
|--------|------|----------|
| 必填校验 | 不能为空 | "请输入目标IP地址或域名" |
| 格式校验 | 合法 IPv4 或域名格式 | "请输入有效的IP地址或域名" |
| 禁用广播 | 不能是 255.255.255.255 | "目标地址不能为广播地址" |

**VRF ID 校验**：

| 校验项 | 规则 | 错误提示 |
|--------|------|----------|
| 必填校验 | 必须选择 | "请选择 VRF 实例" |

#### 3.1.4 交互逻辑

**主流程（Happy Path）**：

```
1. 用户选择 VRF ID
2. 输入目标 IP 地址或域名
3. 点击「Ping」按钮
4. 按钮变为"执行中..."，禁用按钮
```

#### 3.3.3 业务规则与前置条件

**关键概念澄清**：

| 概念 | 英文 | 说明 |
|------|------|------|
| 管理状态 | Admin Status | 端口是否被管理员使能（UP/DOWN），即是否被 shutdown |
| 物理链路状态 | Link/Oper Status | 物理链路是否连通（UP/DOWN），网线是否插好 |

**前置条件（修正后）**：

| 规则 | 描述 | 设计理由 |
|------|------|----------|
| Admin Status 必须为 UP | 端口未被管理员手动 shutdown | 只有使能的端口才能进行检测 |
| Link Status 可为 DOWN | **允许在物理链路断开时执行检测** | VCT 的核心价值就是检测断线！如果 Link 必须为 UP 才能检测，那断线时就没法用了，陷入死锁 |
| 端口类型必须为电口 | 仅支持电口（RJ45），不支持光口 | VCT 基于电信号反射原理，光口不适用 |

**业务场景与状态关系**：

| 场景 | Admin Status | Link Status | 是否可检测 | 检测结果预期 |
|------|--------------|-------------|------------|--------------|
| 正常工作 | UP | UP | ✅ 可检测 | 电缆正常，长度正常 |
| 网线断了 | UP | DOWN | ✅ **可检测** | 显示断路位置（如距离交换机 30m 处断开） |
| 水晶头坏了 | UP | DOWN | ✅ **可检测** | 显示断路或短路 |
| 端口被 shutdown | DOWN | - | ❌ 不可检测 | 提示"端口已关闭，请先启用端口" |
| 光纤端口 | - | - | ❌ 不可检测 | 提示"光口不支持虚拟电缆检测" |

**截图错误提示的正确解读**：

截图显示"只有状态为UP的电口支持虚拟电缆检测"，这里的"UP"指的是 **Admin Status（管理状态）**，而非 Link Status。需与研发确认底层实际机制。

**端口下拉选项过滤**：
- 只显示电口类型端口
- 显示格式：`eth0/1 (电口, UP)`、`eth0/2 (电口, DOWN)`、`光纤端口不显示`

#### 3.3.6 超时与性能约束

| 约束项 | 值 | 说明 |

#### 3.3.7 电缆状态说明

| 状态 | 含义 | 可能原因 |
|------|------|----------|
| 正常 | 电缆连接正常 | - |
| 断路 | 某对线断裂 | 网线损坏、水晶头接触不良 |
| 短路 | 线对之间短路 | 网线破损、水晶头压线错误 |
| 线序错误 | 线序不符合标准 | 水晶头压线顺序错误 |
| 阻抗异常 | 阻抗不匹配 | 网线质量差、长度过长 |
| 交叉连接 | 线对交叉连接 | 可能是交叉线或压线错误 |

## 五、附录

### 5.1 数据字典

| 术语 | 说明 |
|------|------|
| VRF | Virtual Routing and Forwarding，虚拟路由转发 |
| 静态路由 | 管理员手动配置的路由条目 |
| 路由距离 | 管理距离，值越小优先级越高 |
| 下一跳 | 数据包转发的下一个路由器地址 |
| 虚拟电缆检测 | VCT (Virtual Cable Test)，检测网线物理状态 |

### 5.2 错误码定义

| 错误码 | 说明 |
|--------|------|
| 400 | 参数校验失败 |
| 404 | 资源不存在 |
| 409 | 配置冲突（如路由已存在） |
| 500 | 系统内部错误 |

### 5.3 业务错误码明细

| 错误码 | 说明 | 所属模块 |
|--------|------|----------|
| ROUTE_EXISTS | 静态路由已存在 | IP路由 |
| NEXTHOP_UNREACHABLE | 下一跳不可达（递归路由） | IP路由 |
| CHIP_ERROR | 底层芯片下发失败 | IP路由 |
| CONCURRENT_MODIFY | 并发修改冲突 | IP路由 |
| VRF_NOT_FOUND | VRF 实例不存在 | 诊断工具 |
| PORT_ADMIN_DOWN | 端口管理状态为 DOWN | 虚拟电缆检测 |
| PORT_NOT_ELECTRICAL | 端口非电口 | 虚拟电缆检测 |
| CABLE_TEST_FAILED | 电缆检测失败 | 虚拟电缆检测 |
