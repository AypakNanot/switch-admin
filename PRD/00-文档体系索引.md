## 四、架构演进说明

### 4.1 架构变化（对比 PRD v1.0）

| 变化点 | PRD v1.0 | 实际实现 | 原因 |
|--------|----------|---------|------|
| Provider 组织方式 | 扁平目录 `cli/`, `mock/` | 子目录 `config/`, `network/`, `maintenance/`, `diagnostic/` | 模块化更清晰 |
| Diagnostic 模块 | 归属 Network | 独立模块 | 职责分离 |
| 模式切换机制 | 环境变量 | ModeResolver + 数据库持久化 | 支持运行时切换 |
| 异步任务管理 | 未明确 | DiagnosticService 统一管理 | 防止孤儿进程 |

### 4.2 新增机制

```
1. 五层架构
   Handler → Service → Provider 接口 → Provider 实现 → DataModel

2. 策略模式 - 双模式切换
   - Mock 模式：离线测试，使用数据库模拟数据
   - Switch 模式：生产环境，使用真实交换机 CLI

3. 异步任务管理
   - Ping/Traceroute/Cable 任务状态机
   - 定期清理超时任务（60 秒）
   - 防孤儿进程机制

4. Provider 子目录组织
   internal/service/provider/
   ├── config/
   │   ├── cli.go
   │   ├── mock.go
   │   ├── interface.go
   │   └── ext.go
   ├── network/
   ├── maintenance/
   └── diagnostic/
```
