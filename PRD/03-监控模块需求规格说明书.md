## 二、端口状态监控

### 2.1 功能说明

展示交换机所有端口的实时状态，包括链路状态、速率、双工模式等关键信息。

### 2.2 界面布局

**菜单路径**：监控 → 端口状态

```
┌─────────────────────────────────────────────────────────────────┐
│ 端口状态                                              [刷新]    │
├─────────────────────────────────────────────────────────────────┤
│ 端口    │ 状态 │ 链路 │ 速率   │ 双工 │ 描述        │ 操作    │
├─────────────────────────────────────────────────────────────────┤
│ GE1/0/1 │ 启用 │ Up   │ 1000M  │ Full │ Server-A    │ [清零]  │
│ GE1/0/2 │ 启用 │ Down │ -      │ -    │             │ [清零]  │
│ GE1/0/3 │ 禁用 │ Down │ -      │ -    │ Unused      │ [清零]  │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 字段定义

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 端口 | String | 端口标识，格式 GE1/0/1 |
| 状态 | Enum | 管理状态：启用/禁用 |
| 链路 | Enum | 物理链路状态：Up/Down |
| 速率 | String | 当前协商速率：10M/100M/1000M/10G |
| 双工 | Enum | 双工模式：Full/Half |
| 描述 | String | 端口描述/别名 |
| 操作 | Button | 单端口清零按钮 |

### 2.4 筛选功能（简化版）

| 筛选项 | 选项 |
|--------|------|
| 状态 | 全部/启用/禁用 |
| 链路 | 全部/Up/Down |

### 2.5 交互逻辑

| 操作 | 流程 |
|------|------|
| 查看端口状态 | 进入页面自动加载所有端口状态 |
| 筛选端口 | 选择筛选条件 → 自动刷新列表 |
| 单端口清零 | 点击「清零」→ 确认后清零该端口统计 |
| 全局清零 | 页面顶部按钮 → 强警告确认 → 清零所有端口 |

### 2.6 API 设计

```
GET  /api/v1/ports                       # 获取端口列表
POST /api/v1/ports/{port_id}/clear-stats # 清零单个端口统计
POST /api/v1/ports/clear-stats           # 清零所有端口统计
```

**响应示例**：

```json
{
  "code": 200,
  "data": {
    "ports": [
      {
        "id": "GE1/0/1",
        "admin_status": "enable",
        "link_status": "up",
        "speed": "1000Mbps",
        "duplex": "full",
        "description": "Server-A",
        "rx_bytes": "1288490188",
        "tx_bytes": "897581056"
      }
    ]
  }
}
```

## 四、工程约束（保留核心项）

### 4.1 流量统计字段必须使用 64 位无符号整数

**原因**：10G 端口在 3.4 秒内即可传输超过 42 亿字节，32 位整数会溢出。

**要求**：
- 后端：使用 `uint64_t` 类型
- API 传输：超过 JavaScript 安全整数范围的数值使用 String 类型
- 前端：使用 BigInt 解析大数字符串

### 4.2 统计清零采用偏移量机制

**原因**：避免直接影响硬件计数器，防止影响 SNMP 等其他监控系统。

**要求**：
- 记录清零时刻的计数值作为 Offset
- 展示时用"当前硬件值 - Offset"显示相对值
- Offset 存储在 Web 进程内存中，设备重启后自动清零

### 4.3 CSV 导出使用流式写入（如需要导出功能）

**原因**：嵌入式设备内存有限（256MB~512MB），全量加载会导致 OOM。

**要求**：
- 采用 Chunked Transfer Encoding 逐行生成
- 每次读取固定批次（如 1000 条），写入后立即释放内存
- 限制同时进行的导出任务数量（≤ 2 个）
